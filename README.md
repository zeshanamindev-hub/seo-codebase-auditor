# seo-codebase-auditor   

> Honest, **codebase-only** SEO audits for Next.js, Shopify, WordPress, and modern web projects — generated by Claude Code from the evidence in your repository.

A Claude Code community plugin that audits your website/app **source code** for SEO and writes a structured, reproducible report. It reads your real files — metadata, schema, sitemap, robots, canonicals, images, components, templates — and grades what it actually finds.

It will **never** invent traffic, rankings, backlinks, domain authority, AI citations, GA4, or Search Console data. Anything that can't be proven from the codebase is reported as **“Not available from codebase.”**

---

## What it does

- Detects your project type from manifest and signature files.
- Inspects only the files that actually exist in the repo.
- Audits 13 codebase-verifiable SEO areas and scores each with a transparent, weighted model.
- Writes three report files into the project so the audit is shareable and reproducible.
- Produces a prioritized, developer-ready action plan.
- Exports that action plan to a shared **Google Sheet** tracker (priorities, assignees, status) on demand.

## What it audits

- **Technical SEO** — indexability, crawlability, rendering mode, error/page structure
- **Metadata** — titles, descriptions, Open Graph, Twitter cards, robots meta, placeholder/duplicate risks
- **Schema** — JSON-LD: Organization, WebSite, Article/BlogPosting, Product, BreadcrumbList, FAQPage
- **Sitemap** — static, `app/sitemap.ts`, `next-sitemap`, or plugin-generated
- **Robots.txt** — accidental blocking, sitemap declaration, AI crawler rules
- **Canonicals** — implementation, missing/wrong risks, localhost/staging URL leaks
- **Internal links** — nav/footer/content links, empty hrefs, orphans, external `rel` hygiene
- **Image optimization** — `next/image` vs raw `<img>`, alt text, LCP/CLS, lazy loading, remote config
- **Accessibility basics** — headings, semantic HTML, labels, ARIA
- **Performance setup** — static/ISR/caching, font/script loading, third-party scripts, bundle risk
- **Framework SEO** — Next.js, Shopify, and WordPress specific implementation details
- **Content structure** — hierarchy, blog/article structure, freshness signals, topic clusters
- **AI visibility readiness** — entity clarity, author/company signals, citation-friendly structure, `llms.txt`

## What it does NOT audit

This is a **static, codebase-only** tool. It has no crawler, browser, or API access. It does **not** measure or estimate:

- Organic traffic, sessions, pageviews
- Keyword rankings, impressions, clicks, CTR
- Backlinks or referring domains
- Domain authority / domain rating
- AI citation counts or any “AI visibility score”
- Google Search Console data
- GA4 / analytics data
- Real SERP performance or competitor comparisons

Those require live data sources. The plugin marks every such item as **“Not available from codebase.”**

## Supported frameworks

| Detected | Notes |
|---|---|
| **Next.js** | App Router & Pages Router, metadata API, `sitemap.ts`/`robots.ts`, `next/image`, `next/font` |
| **Shopify** | Liquid themes (`theme.liquid`, templates, sections, snippets) and Hydrogen |
| **WordPress** | Classic & block themes (`functions.php`, `wp_head`, `theme.json`, template parts) |
| **Astro / Nuxt / Remix / Gatsby** | Framework-idiomatic head/meta, sitemap, and image checks |
| **Vite / React (SPA)** | Flags client-only rendering and prerender configuration |
| **Static HTML** | Raw `<head>`, links, images, and structured data |
| **Unknown** | Universal checks only; scores honestly as `N/A` where evidence is thin |

## Installation

Clone or download this plugin, then point Claude Code at it:

```bash
claude --plugin-dir ./seo-codebase-auditor
```

You can also place it in your Claude Code plugins directory or install it from a marketplace that lists it.

## Local usage

1. Open Claude Code **inside the repository you want to audit**.
2. Load the plugin (see installation).
3. Run the command:

```text
/seo-codebase-auditor:audit
```

### Example prompt

For a fully explicit, honesty-first run, paste:

```text
/seo-codebase-auditor:audit
Audit this project codebase only. Do not invent traffic, keywords, backlinks, domain authority, or AI visibility rankings. Generate reports/codebase-seo-audit.md, reports/codebase-seo-action-plan.md, and reports/codebase-seo-evidence.json.
```

## Output files

The audit writes three files into the audited project:

| File | Purpose |
|---|---|
| `reports/codebase-seo-audit.md` | Full 18-section report with scores, grades, evidence, and per-category findings |
| `reports/codebase-seo-action-plan.md` | Prioritized fixes (High/Medium/Low) ranked by `Impact + Confidence − Effort` |
| `reports/codebase-seo-evidence.json` | Machine-readable evidence: files reviewed, files missing, per-category scores, and an explicit `notAvailableFromCodebase` list |

See [`examples/sample-codebase-audit.md`](examples/sample-codebase-audit.md) for a realistic, filled-in example (fictional Next.js project) and [`templates/codebase-seo-audit-report.md`](templates/codebase-seo-audit-report.md) for the blank report template.

The `codebase-seo-evidence.json` also carries a machine-readable `actionPlan` array (one object per task), which powers the Google Sheet export below.

## Export the action plan to a Google Sheet

After running the audit, you can turn the action plan into a shared, formatted
**Google Sheet** project tracker — on demand — via the `sheet` skill:

```text
/seo-codebase-auditor:sheet
```

or just ask: *"export the SEO action plan to a Google Sheet."* The skill will ask
for an optional **team roster**, an **assignment mode** (round-robin or leave
blank), and the **email to share the sheet with**, then create:

- a **Summary** tab — task counts by priority/status/assignee and the category grades, and
- a **Task Tracker** tab — one row per task with Bucket, Priority, Impact/Effort/Confidence, Files to update, plus **Assignee** and **Status** dropdowns and color-coded priority buckets.

The new spreadsheet's URL is printed and saved to `reports/codebase-seo-sheet-url.txt`.

**Honesty preserved:** tasks and scores come only from the audit; assignees come
only from the roster you provide (never invented), and the Summary keeps the
codebase-only disclaimer.

### One-time setup

Sheet creation uses a Google **service account** (headless, no browser). Follow
[`docs/google-sheets-setup.md`](docs/google-sheets-setup.md) once: enable the
Google Sheets + Drive APIs, create a service account, download its JSON key, and
set `SEO_SHEET_SA_KEY` to the key's path. No `npm install` is needed — the
builder ([`scripts/build-sheet.mjs`](scripts/build-sheet.mjs)) is dependency-free.

No credentials yet? Preview the output with no setup and no network calls:

```bash
node scripts/build-sheet.mjs --input reports/codebase-seo-evidence.json --dry-run
```

This writes `reports/codebase-seo-tasks-preview.csv` (importable into any
spreadsheet) plus the exact API payload it *would* send.

## Scoring

Each category is scored 0–100 from weighted sub-criteria and mapped to a grade:

| Score | Grade |
|---:|---|
| 90–100 | A |
| 80–89 | B |
| 70–79 | C |
| 60–69 | D |
| 0–59 | F |
| N/A | Not enough codebase evidence |

Every score is traceable to a file. Each category also carries a **Confidence** rating (High/Medium/Low) reflecting how directly the evidence was verified. The full weight breakdown lives in [`skills/audit/SKILL.md`](skills/audit/SKILL.md).

## Evidence-first audit policy

This plugin is built around one rule: **if it isn't in the code, it isn't in the report.**

- Every claim traces to a real file path.
- Unverifiable metrics are labeled **“Not available from codebase,”** never guessed.
- `N/A` is a valid, honest result — scores are not padded to look complete.
- Confidence ratings keep speculative findings clearly marked.
- No external APIs, no network calls, no analytics integrations.

This makes the audit reproducible, reviewable, and safe to share with clients or teammates without overclaiming.

## Community / marketplace positioning

`seo-codebase-auditor` fills a specific gap: most SEO tools need a live URL and paid data sources. This one runs **entirely on your repository**, before deploy, in CI, or during a client codebase review — and stays honest about its limits.

- **For developers:** ship-blocking issues (blocked robots, localhost canonicals, missing schema) caught at the code level.
- **For agencies/freelancers:** a clean, evidence-backed deliverable generated from a codebase you were handed.
- **For teams:** a repeatable pre-launch SEO checklist that lives next to the code.

Pairs naturally with live-data SEO tools — use this for the implementation layer, and your analytics/Search Console stack for performance.
